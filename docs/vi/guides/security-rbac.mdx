---
title: "Bảo mật và RBAC"
description: "Bảo mật HTTP deployment bằng API keys và role-based access control."
---

Với HTTP deployment, hãy bật API-key auth và phân quyền theo vai trò.

<Warning>
  Không public HTTP transport nếu chưa bật `ENABLE_AUTH=true`.
</Warning>

## Bật auth

```bash
ENABLE_AUTH=true
AUTH_HEADER_NAME=x-api-key
AUTH_API_KEYS=viewer-key:viewer:viewer1,editor-key:editor:editor1,admin-key:admin:admin1
```

Client phải gửi auth header đã cấu hình:

```http
x-api-key: editor-key
```

## Vai trò

| Vai trò | Quyền hạn |
|---------|-----------|
| `viewer` | Thao tác đọc (list, read, search, status) |
| `editor` | Quyền viewer + ghi không phá hủy, build và test |
| `admin` | Toàn quyền, bao gồm thao tác phá hủy (delete, restore, force push) |

## Kiểm soát bảo mật trong codebase

| Kiểm soát | Mô tả |
|-----------|-------|
| Chống path traversal | Mọi path đều được resolve trong boundary root của project |
| Chặn protected-path | Các `PROTECTED_PATHS` đã cấu hình không thể bị mutation |
| Confirm flags | Thao tác phá hủy yêu cầu `confirm: true` tường minh |
| Command allowlist | Chỉ lệnh trong `ALLOWED_COMMANDS` mới được thực thi |
| Giới hạn kích thước output | Output của command và resource bị cắt tại `MAX_OUTPUT_SIZE` |
| Khóa theo project | Thao tác mutation dùng lock theo từng project để tránh race condition |

## Khuyến nghị vận hành

- Luân phiên API keys định kỳ
- Dùng key riêng cho từng môi trường
- Giới hạn admin keys cho automation đáng tin cậy
- Bật telemetry để audit request
