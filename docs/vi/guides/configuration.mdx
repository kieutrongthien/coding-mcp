---
title: "Cấu hình"
description: "Cấu hình coding-mcp bằng biến môi trường hoặc file cấu hình."
---

Bạn có thể dùng `.env`, biến môi trường, hoặc `MCP_CONFIG_PATH` với file JSON/YAML.

## Thiết lập cốt lõi

| Biến | Mô tả |
|------|-------|
| `PROJECTS_ROOTS` | Danh sách thư mục root phân tách bằng dấu phẩy (khuyến nghị). |
| `PROJECTS_ROOT` | Một thư mục root duy nhất (tương thích legacy). |
| `ENABLE_HTTP` | Bật HTTP transport (`true`/`false`). |
| `ENABLE_STDIO` | Bật STDIO transport (`true`/`false`). |
| `HTTP_HOST` | Địa chỉ bind của HTTP server. Mặc định: `127.0.0.1`. |
| `HTTP_PORT` | Cổng của HTTP server. Mặc định: `3000`. |
| `HTTP_MODE` | Chế độ HTTP transport: `streamable` hoặc `sse`. |
| `MAX_FILE_SIZE` | Kích thước file tối đa cho thao tác đọc (bytes). |
| `MAX_OUTPUT_SIZE` | Kích thước output tối đa cho command hoặc resource (bytes). |
| `COMMAND_TIMEOUT_MS` | Timeout cho việc thực thi command (milliseconds). |
| `RETRY_MAX_ATTEMPTS` | Số lần retry khi gặp lỗi tạm thời. |
| `RETRY_BASE_DELAY_MS` | Thời gian chờ cơ bản giữa các lần retry (milliseconds). |
| `ALLOWED_COMMANDS` | Danh sách lệnh cho phép bởi `run_command_safe`, phân tách bằng dấu phẩy. |
| `PROTECTED_PATHS` | Pattern đường dẫn không thể sửa đổi, phân tách bằng dấu phẩy. |
| `LOG_LEVEL` | Mức độ chi tiết log: `trace`, `debug`, `info`, `warn`, `error`. |
| `REGISTRY_FILE` | Đường dẫn đến file JSON registry dự án. |
| `AUDIT_LOG_FILE` | Đường dẫn đến audit log cho các thao tác có mutation. |
| `HTTP_REQUEST_LOG_FILE` | Đường dẫn đến HTTP request log. |

## Thiết lập bảo mật

| Biến | Mô tả |
|------|-------|
| `ENABLE_AUTH` | Bật xác thực bằng API key cho HTTP transport. |
| `AUTH_HEADER_NAME` | HTTP header dùng để truyền API key. Mặc định: `x-api-key`. |
| `AUTH_API_KEYS` | Danh sách keys phân tách bằng dấu phẩy theo định dạng `key:role:id`. |

## Thiết lập quan sát hệ thống

| Biến | Mô tả |
|------|-------|
| `ENABLE_OTEL` | Bật OpenTelemetry tracing. |
| `OTEL_SERVICE_NAME` | Tên service báo cáo trong traces. |
| `OTEL_EXPORTER_OTLP_ENDPOINT` | OTLP HTTP endpoint để xuất trace. |
| `OTEL_EXPORTER_OTLP_HEADERS` | Headers cho OTLP exporter theo định dạng `key=value,key2=value2`. |

## Đường dẫn đa nền tảng

`REGISTRY_FILE` và `AUDIT_LOG_FILE` hỗ trợ `~` và được chuẩn hóa thành đường dẫn tuyệt đối trên Linux, macOS và Windows.

## Ví dụ `.env`

```bash
PROJECTS_ROOTS=/projects,/srv/repos
ENABLE_HTTP=true
ENABLE_STDIO=true
HTTP_HOST=127.0.0.1
HTTP_PORT=3000
HTTP_MODE=streamable
MAX_FILE_SIZE=1048576
MAX_OUTPUT_SIZE=131072
COMMAND_TIMEOUT_MS=120000
RETRY_MAX_ATTEMPTS=2
RETRY_BASE_DELAY_MS=200
ALLOWED_COMMANDS=npm,pnpm,yarn,go,python,pytest,ruff,flake8,cargo,mvn,gradle,php,composer,phpunit,make
PROTECTED_PATHS=.git,.env,.env.local,.env.production
LOG_LEVEL=info
REGISTRY_FILE=~/.coding-mcp/registry.json
AUDIT_LOG_FILE=~/.coding-mcp/audit.log
HTTP_REQUEST_LOG_FILE=~/.coding-mcp/http-requests.log
```

## Dùng file cấu hình

```bash
MCP_CONFIG_PATH=/etc/coding-mcp/config.yaml coding-mcp serve --transport http
```

## Validation

Nếu cấu hình không hợp lệ, quá trình khởi động sẽ dừng với schema validation errors.
