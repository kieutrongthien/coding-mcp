`coding-mcp` cung cấp 3 lớp quan sát:

- Structured logs (Pino)
- Audit logs cho thao tác có mutation
- OpenTelemetry trace hooks

## Logging

Đặt `LOG_LEVEL` để điều khiển mức chi tiết.

```bash
LOG_LEVEL=info
```

## Audit log

Các thao tác có mutation được ghi thêm vào file audit log.

```bash
AUDIT_LOG_FILE=~/.coding-mcp/audit.log
```

## OpenTelemetry

Bật OTLP export:

```bash
ENABLE_OTEL=true
OTEL_SERVICE_NAME=coding-mcp
OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318/v1/traces
OTEL_EXPORTER_OTLP_HEADERS=authorization=Bearer%20token
```

Spans được phát:

| Span | Mô tả |
|------|-------|
| `mcp.http.request` | Phát cho mỗi HTTP request mà server nhận |
| `mcp.tool.{operation}` | Phát cho mỗi lần thực thi tool, đặt tên theo operation |

## HTTP monitoring endpoints

Khi bật HTTP transport:

| Endpoint | Mô tả |
|----------|-------|
| `GET /healthz` | Liveness check — trả về `ok` khi server đang chạy |
| `GET /readyz` | Readiness check — trả về `ok` kèm số lượng project đã index |
| `GET /metrics` | Metrics cơ bản: `request_count`, `error_count` và latency summary |

## Xử lý sự cố

Nếu traces không xuất hiện:

1. Kiểm tra khả năng truy cập exporter endpoint
2. Kiểm tra định dạng OTLP headers (`key=value,key2=value2`)
3. Kiểm tra server logs để tìm lỗi khởi tạo
