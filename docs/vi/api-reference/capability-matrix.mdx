Dùng trang này như tài liệu tra cứu nhanh cho tool contracts và yêu cầu phân quyền.

## Quản lý dự án

| Tool | Key inputs | Mutating | Notes |
| --- | --- | --- | --- |
| `list_projects` | none | No | Liệt kê các dự án đã index |
| `get_project` | `project_id` | No | Trả về tóm tắt metadata |
| `refresh_project_index` | `project_id?` | Yes | Build lại index cho một/tất cả dự án |
| `summarize_project` | `project_id` | No | Tóm tắt cấp cao cho dự án |

## File và patch tools

| Tool | Key inputs | Mutating | Safety controls |
| --- | --- | --- | --- |
| `list_directory` | `project_id`, `path` | No | Root boundary checks |
| `read_file` | `project_id`, `path`, `start_line?`, `end_line?` | No | Max file size limit |
| `read_multiple_files` | `project_id`, `paths[]` | No | Per-file size validation |
| `create_file` | `project_id`, `path`, `content`, `overwrite?` | Yes | Protected path checks |
| `write_file` | `project_id`, `path`, `content` | Yes | Protected path checks |
| `replace_in_file` | `project_id`, `path`, `find`, `replace`, `replace_all?` | Yes | Protected path checks |
| `apply_patch` | `project_id`, `path`, `patch?`, `structured_edits?`, `dry_run?` | Yes | Hunk conflict reporting, dry run |
| `delete_file` | `project_id`, `path`, `recursive?`, `confirm` | Yes | Yêu cầu `confirm=true` |
| `move_file` | `project_id`, `source_path`, `target_path` | Yes | Source/target boundary checks |

## Mô hình vai trò (HTTP mode)

| Role | Access |
| --- | --- |
| `viewer` | Thao tác thiên về đọc |
| `editor` | Viewer + workflows ghi/build/test không phá hủy |
| `admin` | Toàn bộ thao tác |

<Note>
  Trong HTTP deployment có bật auth, role checks chạy trước mỗi tool operation. Ở STDIO mode, mô hình tin cậy local process được áp dụng nếu không có wrapper bổ sung.
</Note>
