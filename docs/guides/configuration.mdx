---
title: "Configuration"
description: "Configure coding-mcp with environment variables or config files."
---

Use `.env`, environment variables, or `MCP_CONFIG_PATH` with a JSON/YAML file.

## Core settings

- `PROJECTS_ROOTS`: comma-separated project roots (preferred)
- `PROJECTS_ROOT`: legacy single-root fallback
- `ENABLE_HTTP`, `ENABLE_STDIO`
- `HTTP_HOST`, `HTTP_PORT`, `HTTP_MODE`
- `MAX_FILE_SIZE`, `MAX_OUTPUT_SIZE`
- `COMMAND_TIMEOUT_MS`
- `ALLOWED_COMMANDS`
- `PROTECTED_PATHS`
- `LOG_LEVEL`
- `REGISTRY_FILE`, `AUDIT_LOG_FILE`

## Cross-platform paths

`REGISTRY_FILE` and `AUDIT_LOG_FILE` support `~` and normalize to absolute paths on Linux, macOS, and Windows.

## Example `.env`

```bash
PROJECTS_ROOTS=/projects,/srv/repos
ENABLE_HTTP=true
ENABLE_STDIO=true
HTTP_HOST=0.0.0.0
HTTP_PORT=3000
HTTP_MODE=streamable
MAX_FILE_SIZE=1048576
MAX_OUTPUT_SIZE=131072
COMMAND_TIMEOUT_MS=120000
ALLOWED_COMMANDS=npm,pnpm,yarn,go,python,pytest,php,composer,make
PROTECTED_PATHS=.git,.env,.env.local,.env.production
LOG_LEVEL=info
REGISTRY_FILE=~/.coding-mcp/registry.json
AUDIT_LOG_FILE=~/.coding-mcp/audit.log
```

## Config file usage

```bash
MCP_CONFIG_PATH=/etc/coding-mcp/config.yaml coding-mcp serve --transport http
```

## Validation

If configuration is invalid, startup fails with schema validation errors.
